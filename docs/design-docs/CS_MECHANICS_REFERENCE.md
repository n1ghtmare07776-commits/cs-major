## 一、核心不对称性：T 方 vs CT 方

CS 的根本魅力来自**攻守不对称**。两支队伍在同一张地图上执行完全不同的任务，拥有不同的信息优势，面临不同的决策压力。

### 1.1 阵营定义

| 维度 | 恐怖分子 / T 方（进攻方） | 反恐精英 / CT 方（防守方） |
|---|---|---|
| **核心任务** | 安装 C4 炸弹并引爆 | 阻止安装，或拆除已安装的 C4 |
| **胜利条件** | A) 炆炸目标点  B) 全歼 CT  C) 时间耗尽时炸弹已安装 | A) 拆除 C4  B) 全歼 T  C) 时间耗尽且 C4 未安装 |
| **时间压力** | 有——必须在回合计时内完成进攻（通常 1:45） | 无——时间本身就是盟友，拖得越久对 CT 越有利 |
| **空间主动权** | **拥有**——选择打哪个包点、何时打、怎么打 | **没有**——必须覆盖两个包点，被动应对 T 的动向 |
| **信息优势** | 进攻意图只有自己知道；可以佯攻欺骗 | 可以利用地图结构获取位置信息；但不知道 T 打哪里 |
| **经济压力** | 更大——需要步枪/狙击枪才能有效进攻 | 较小——防守方可以用更便宜的武器发挥价值（如沙鹰+闪光） |
| **装备依赖** | 高——eco 局几乎无进攻能力 | 中——eco 局仍可依靠地形和道具进行防守 |

### 1.2 不对称性对游戏设计的含义

**关键原则**：T 方和 CT 方的"好选项"是**不同集合**。

- 给 T 方展示"架枪守包点"选项 = 无意义（T 方没有包点可守）
- 给 CT 方展示"快攻下包"选项 = 语义混乱（CT 方不下包）
- 两方的决策词汇、战术思维模型、成功标准完全不同

## 二、回合结构与时序

### 2.1 标准回合时间线

```
┌─────────────────────────────────────────────────────────────┐
│  回合开始                                                    │
│  ├── 购买阶段 (freeze time, ~15s)                            │
│  │   双方决定本局经济投入                                      │
│  │   选择武器、护甲、道具                                     │
│  ├───── 移动阶段 ────────────────────────────────────────    │
│  │   T 方：选择进攻路线和节奏                                  │
│  │   CT 方：选择站位和开局策略                                │
│  ├───── 交火阶段 ────────────────────────────────────────    │
│  │   信息收集 → 接触 → 减员 → 控图/缩圈                       │
│  ├───── 关键节点 ────────────────────────────────────────    │
│  │   [如果 T 成功进入包点]                                    │
│  │   ├── 下包（plant the bomb）                               │
│  │   └── 进入残局/回防阶段                                    │
│  │   [如果 CT 成功阻止或全歼 T]                               │
│  │   └── 回合结束                                            │
│  ├───── C4 倒计时（仅当炸弹已安装）───────────────────────     │
│  │   40 秒倒计时                                             │
│  │   T 方：守包防止拆弹                                       │
│  │   CT 方：组织回防拆弹                                      │
│  └── 回合结束                                                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 回合中的关键决策时刻

| 时刻 | T 方决策焦点 | CT 方决策焦点 |
|---|---|---|
| **购买后** | 进攻哪个包点？快还是慢？ | 重防还是前压？堆人还是分散？ |
| **接触前** | 是否假打？是否转点？是否单人摸排？ | 是否换位？是否丢道具拖延？ |
| **首杀后** | 趁势进点还是继续控图？ | 补枪还是保枪？缩圈还是反清？ |
| **减员后（3v4 等）** | 强行执行还是拖时间？ | 反压还是稳守？ |
| **进入包点后** | 快下包还是搜点清完？ | 直接回防还是先断后路？ |
| **下包后** | 怎么架枪？谁看哪个角度？ | 从哪路回？怎么拆？ |
| **残局（≤3v3）** | 架枪还是找人对枪？ | 拆弹还是找人？ |

## 三、经济系统详解（真实规则还原）

> **核心原则**: CS 的经济不是"每局自由选"。所有金钱从 **$800 手枪局** 起步，通过轮次胜负、击杀、连败奖励逐步累积。玩家不能凭空选择"全买"——必须先有钱。

### 3.0 经济基础数值速查

| 项目 | 数值 | 说明 |
|---|---|---|
| **起始金钱** | $800 | 每半场 / 加时首局 |
| **最大金钱** | $16,000 | 金钱硬上限 |
| **赢局奖励** | $3,250（淘汰全队）/$3,500（炸蛋/拆弹） | 赢下完整回合 |
| **步枪/手枪击杀奖励** | $300 | M4/AK/Galil/FAMAS/沙鹰/USP 等 |
| **冲锋枪击杀奖励** | $600 | MP9/MAC-10/MP7/UMP/P90 等 |
| **霰弹枪击杀奖励** | $900 | MAG-7/NOVA/XM1014 等 |
| **AWP 击杀奖励** | $100 | — |
| **刀杀奖励** | $1,500 | — |
| **C4 安放奖励（T）** | $300（个人，无论输赢） | 只要炸弹成功下包就有 |
| **连败奖励递增** | $1,400 → $1,900 → $2,400 → $2,900 → $3,400 | 每次连败递增 $500 |
| **连败重置条件** | 赢下一局 | 赢后连败计数归零 |
| **存活到下局不买** | 保留上局装备 | 不花钱就是省 |

### 3.1 手枪局 — 强制起始点

**非可选。任何比赛 / 任何半场的第一局都是手枪局。**

| 维度 | 说明 |
|---|---|
| **金钱** | 双方各 $800，没有例外 |
| **可买装备** | 强化手枪（P250 / Deagle / Tec-9 / Five-SeveN）+ 护甲（$650） + 手雷（可选）+ CT 可选拆弹套件（$400） |
| **不可买装备** | 步枪、冲锋枪（买不起或买不起甲）、AWP、M4、AK 等 |
| **玩家决策** | 战斗用品还是全甲？是否买道具/拆弹器？偏进攻方向还是保守消耗？ |
| **策略意义** | 手枪局的结果直接影响接下来 2-4 局的经济走向——这是整场比赛最重要的单一回合 |

### 3.2 经济等级（Buy Tier）——不是"可选菜单"

这些等级**不是玩家自由选择的按钮**。它们是**当前累计金钱所允许的装备水平**。玩家可以在允许范围内微调比重（多买道具少买甲 vs 全甲少道具），但不能跨等级消费。

| 等级 | 名称 | 所需金钱（人均） | 典型配置 | 战斗力 | 何时出现 |
|---|---|---|---|---|---|
| **Lv5** | **全买（Full Buy）** | $4,000+ | 步枪×4 + AWP×1 + 全甲 + 满道具 + 拆弹器 | 100% | 第 3-4 局以后，需连胜或连败累计 |
| **Lv4** | **半买（Half-Buy）/ 强起（Force Buy）** | $2,700-3,500 | 步枪×2-3 + 冲锋枪 + 半甲 + 少量道具 | 65-75% | 第 2 局输手枪方；或输几局后搏一把 |
| **Lv3** | **弱起（Light Buy）** | $1,800-2,500 | 冲锋枪/霰弹枪 + 甲 + 手雷×1-2 | 40-50% | 连输两局后的过渡 |
| **Lv2** | **ECO（纯经济局）** | $0-1,200 | 沙鹰+无甲，或 P250+半甲，或纯原始手枪 | 20-30% | 第 2 局输手枪方；或任何"这局白给，下局打回来" |
| **Lv1** | **纯 ECO / 全保** | $0-500 | 只有开局手枪，什么都不买 | 10-15% | 极少数情况（完全没钱且不想冒险） |

### 3.3 完整经济进度：5 局比赛的标准流动

以下展示一支队伍在 BO5 中最常见的**两种经济轨迹**。标注的购买等级是**当前金钱实际允许的**，不是玩家自由选择的。

#### 轨迹 A：赢手枪局 → 滚雪球

```
第 1 局 [手枪局]  $800 → 只能手枪
  → 赢！ +$3,250 回合奖金 + 击杀奖励
  ─────────────────────────────────
第 2 局           ~$4,000-5,000 → Lv4-Lv5 ★ 可以冲锋枪/少量步枪
  → 赢！（anti-eco 大概率赢） +$3,250+ 击杀
  ─────────────────────────────────
第 3 局           ~$5,000-7,500 → Lv5 全买 ★ 可以AK/AWP+全甲+满道具
                  或 → 保留上局冲锋枪，轻买 → **Bonus Round** 省钱局
  → 赢/输
  ─────────────────────────────────
第 4 局           根据上局结果：Lv5 全买 或 Lv4 半买
第 5 局           大概率 Lv5（赢了有钱，连输也有连败奖励）
```

#### 轨迹 B：输手枪局 → 挣扎追回

```
第 1 局 [手枪局]  $800 → 只能手枪
  → 输！ +$1,400 连败奖励①（可能 + 少数击杀）
  ─────────────────────────────────
第 2 局           ~$1,400-1,900 → 两个选择：
                  A) Lv2 ECO：剩钱买沙鹰或全保  → 目标："这局白送，第 3 局打回来"
                  B) Lv4 强起：最大化当期输出 → 赌一把翻盘
                  → 大概率输（装备差）
  → 输！ +$1,900 连败奖励②
  ─────────────────────────────────
第 3 局           ~$2,400-3,300 + 上局剩下的钱
                  如果第 2 局纯 ECO：≈$3,400-4,300 → Lv4-Lv5 ★ 第一次有真正火力！
                  如果第 2 局强起了：≈$1,900-2,800 → Lv3-Lv4，勉强能打
  → 这局是关键翻盘点
  ─────────────────────────────────
```

**关键洞察**：轨迹 A 在第 3 局出现 **Bonus Round** 选项（省钱留冲锋枪，不买步枪，为第 4 局省出两个 AWP）。轨迹 B 在第 3 局到达第一个有战斗力的经济点。真实 CS 的经济节奏本质上是"每 3 局一个周期"。

### 3.4 连败奖励机制

> 连败奖励是 CS 经济最核心的反雪球机制：连续输的队拿的钱逐局递增，自然在某个点恢复战斗力。

| 连续输局数 | 当轮奖励 | 残留金钱 + 可能的击杀奖励 | 约可达到等级 |
|---|---|---|---|
| 1 | +$1,400 | ~$1,400-1,900 | LV2 ECO 或强制 |
| 2 | +$1,900 | ~$2,400-3,300 | LV3-LV4 |
| 3 | +$2,400 | ~$3,900-4,800 | LV4-LV5 |
| 4+ | +$2,900-3,400 | ~$5,000+ | LV5 全买 |

**设计含义**：在经理游戏中，这意味著不存在"输了一直穷"的死循环。连输 3 局后经济自然恢复，真正的管理挑战是这 3 局里怎么打。

### 3.5 Bonus Round（省钱局）

当一个队伍赢了多局、装备不错但不想一口气清空存款时，会出现 Bonus Round 选项：

- **触发条件**: 第 3-4 局 + 有足够的存款 + 上局装备还有利用价值
- **决策逻辑**: "这局不买步枪，用上局的冲锋枪打，赢了白赚，输了也有钱下局全买"
- **在经理游戏中的表达**: 显示为 Lv3-Lv4 等级 + 额外存款 + 标签 "下局可全买"

### 3.6 经济对战术范围的影响

**同样的战术选择，不同经济下可行性完全不同**：

| 可用金钱 | T 方可行战术 | CT 方可行战术 |
|---|---|---|
| $4,000+ Lv5 | 全部（rush 因火力优势更强） | 全部（前压因火力安全更可行） |
| $2,700-3,500 Lv4 | Default/Lurk/Fake 可行，Rush 略弱 | Retake/Stack 可行，Aggro 风险高 |
| $1,800-2,500 Lv3 | 只能 Default（慢搜）或赌命 Rush | 只能保守回防或赌命 Stack |
| $0-1,200 Lv2-ECO | **无进攻能力**。目标：不死保枪 + 看能不能偷一个 | **只能赌 Stack 或赌前压偷人**。目标：杀了就是赚 |
| 手枪局 | 手枪局独有的决策集（见 3.1） | 同左 |

**Lv2 ECO 不是"什么都做不了"**。ECO 回合的微型博弈是经理感的关键：
- "要不要买一个烟雾弹配合沙鹰偷一个？"
- "要不要全体 rush 一个点赌对面失误？"
- "要不要纯保，活着把现有手枪留到下局？"

## 三-A、道具系统（Utility）

### 道具类型及其战术功能

| 道具 | 价格 | 功能 | T 方主要用法 | CT 方主要用法 |
|---|---|---|---|---|
| **烟雾弹（Smoke）** | $300 | 封锁视线 ~18 秒 | 封 CT 架枪视线以便下包；封回防路线 | 封 T 进攻路线拖延时间；下包后封包点 |
| **闪光弹（Flash）** | $200 | 致盲范围内看到的人 ~2 秒 | 配合快攻 blind 冲点；清角用 | 反闪退敌；回防时 blind 进点的 T |
| **燃烧弹**（T: 燃烧瓶 / CT: 燃烧手雷）| $400 / $600 | 对区域造成持续伤害 ~7 秒 | 烧 CT 躲藏的死角（烧脚）；逼 CT 移位 | 烧 T 可能藏的位置；烧包点区域拖延拆弹 |
| **高爆弹（HE）** | $300 | 爆炸范围伤害 | 炸聚集的 CT；炸已知位置的人 | 炸下包后的 T |
| **诱饵弹（Decoy）** | $50 | 模拟开火声音 | 假装在这个方向进攻 | 假装这里有人在 |
| **拆弹套件（CT 专属）** | $400 | 5 秒拆弹（无 kit 需 10 秒） | / | 必买项之一 |

### 道具在经济中的实际权重

- **手枪局**: 可能 0-1 个道具（常省一个闪光或烟雾）
- **全买局**: 人均 2-4 个道具（烟雾必带 x1-2，闪光 x1，燃烧/HE 按需）
- **ECO 局**: 1 个或完全不用；有时全队凑钱买一个烟雾做 surprise play

### 道具在经理游戏中的抽象化

不需要模拟每个道具的使用。建议用 **"道具强度修饰符"** 表达：

| 可支配道具水平 | 修饰符 | 含义 |
|---|---|---|
| 满道具 | +3 道具优势 | 可执行所有复杂战术（多烟雾封锁 + 闪光配合 + 燃烧排点） |
| 基础道具 | +1~2 道具优势 | 有基础配合能力 |
| 缺乏道具 | 0 | 没有足够的道具影响局势 |
| 无道具 | -1~2 道具劣势 | 缺少道具无法执行复杂战术，只能靠枪法 |

## 四、T 方（恐怖分子/进攻方）战术全景

### 4.1 T 方战术分类总览

```
T 方战术按"节奏 × 投入"二维划分：

          低控图投入          高控图投入
        ┌──────────┬──────────┐
 快节奏  │ Rush     │ Fast Ex- │
        │ 快攻      │ ecute    │
        │          │ 快速执行  │
        ├──────────┼──────────┤
 慢节奏  │ Slow     │ Default  │
        │ Lurk     │ 默认控图  │
        │ 单摸/阴人 │          │
        └──────────┴──────────┘
```

### 4.2 T 方核心战术详述

#### 战术 A：Rush（快攻 / 爆弹抢点）

- **描述**: 全队集中兵力在短时间内从一个方向猛攻一个包点
- **典型执行**: 5 人同时扔烟雾弹+闪光弹→冲进包点→快速下包
- **优势**: 以多打少（局部人数优势），不给 CT 反应时间
- **劣势**: 如果被 CT 提前读到（stack/重防），会遭遇交叉火力团灭；一旦失败经济损失巨大（道具全用了）
- **适合场景**: 对方防线松散、己方经济好需要快速拿分、对方习惯慢开局的时刻
- **克制手段**: CT 堆人重防（Stack Site）、CT 前压抓 timing
- **所需属性**: firepower > tactics > discipline
- **失败后果**: 团灭 + 道具全空 + 经济崩溃

#### 战术 B：Default（默认控图 / 默认架枪）

- **描述**: 缓慢推进，逐步控制地图中路和连接区域，收集信息后再决定进攻哪个点
- **典型执行**: 3 人控制中路，2 人分别向两个包点试探性施压；根据 CT 防守反应决定最终主攻方向
- **优势**: 信息充分后做决策，容错率高；可以根据 CT 的换位调整
- **劣势**: 耗时长（可能被 CT 抓到单摸的人）；需要良好的沟通和纪律
- **适合场景**: 己方战术执行力强、想稳妥拿分、对手喜欢前压被抓
- **克制手段**: CT 前压打断控图节奏、CT Info Control 拖延时间
- **所需属性**: tactics > discipline > cohesion
- **失败后果**: 时间不够被迫仓促进点

#### 战术 C：Slow Lurk（慢控单摸 / 阴人）

- **描述**: 4 人在一侧制造压力/假装进攻，1 人（lurker）悄悄绕到另一个包点后方等待机会
- **典型执行**: 4 人 A 点佯攻/控图，lurker 从 B 点后方静步接近，等 CT 回防 A 时从背后偷人
- **优势**: 制造人数错觉，可能打出"偷背身"的多杀；心理压力极大
- **劣势**: lurker 孤立无援如果被提前发现就白给；非常依赖 lurk 个人能力
- **适合场景**: 对方防守偏重一边、对手有明显的回防习惯、lurker 选手状态好
- **克制手段**: CT 保守站位不留后路、CT 注意听脚步/检查盲区
- **所需属性**: clutch > tactics > discipline
- **失败后果**: lurker 白给导致 4v5

#### 战术 D：Fake / Rotate（假打转点）

- **描述**: 在一个包点制造大规模进攻假象（扔道具、出声音、露头），吸引 CT 重防该点，然后主力转向另一个包点
- **典型执行**: 3-4 人在 A 点疯狂扔烟雾闪光制造声势（甚至假装冲），实际 2-3 人打空无一人的 B 点
- **优势**: 如果 CT 上当了，进攻方是多人打少人或无人；收益极高
- **劣势**: 如果 CT 没上当（没换防），进攻方等于分散兵力被打各个击破
- **适合场景**: 对方是"看到动静就回防"的类型、侦察显示对方某个选手容易慌乱换位
- **克制手段**: CT 不轻易换防、CT 通过信息确认真伪再动
- **所需属性**: tactics > cohesion > discipline
- **失败后果**: 兵力分散被逐个击破

#### 战术 E：Late Game Execute（拖后期执行）

- **描述**: 前大部分时间缓慢控图收集信息，最后 20-25 秒突然快速打进选定的包点
- **典型执行**: 前 80 秒慢慢摸、试探、换位；最后 20 秒全员集结一个点强行突破
- **优势**: 信息最充分，CT 的防守布置已经被摸清；时间压力迫使 CT 不能再换位
- **劣势**: 需要极高的执行力和时间管理能力；万一拖太久了来不及下包
- **适合场景**: 己方战术素养极高、想打对手一个时间差
- **克制手段**: CT 稳守不乱动、CT 在最后时刻精准预判进攻点
- **所需属性**: tactics > discipline > cohesion

### 4.3 T 方特殊情境战术

#### 下包后架枪（Post-Plant Hold）

这是 T 方独有的情境——**炸弹已经安放，角色从"进攻者"变成了"防守者"**。

- **T 方任务**: 阻止 CT 在 40 秒内拆弹
- **基本思路**: 利用炸弹爆炸的声音/位置信息作为诱饵，在拆弹点周围架枪
- **常见架枪方式**:
  - 1 人直接看住包点（近距离）
  - 其他人分散在不同角度形成交叉火力网
  - 有人专门负责"补枪"（队友被杀后补位）
- **关键变量**:
  - 下包位置（好位置让 CT 多花 1-2 秒到达 = 生死之差）
  - 剩余 CT 人数和位置
  - 自身剩余血量和道具
  - 拆弹器是否有（有拆弹器 = 5 秒拆除；无 = 10 秒拆除）

## 五、CT 方（反恐精英/防守方）战术全景

### 5.1 CT 方战术分类总览

```
CT 方战术按"防守姿态 × 主动性"划分：

          被动/反应型           主动/干扰型
        ┌──────────┬──────────┐
 站位保守│ Conservative│ Stack    │
        │ Retake    │ 重防堆人  │
        │ 保守回防   │ 赌点     │
        ├──────────┼──────────┤
 站位积极│ Info      │ Early Agg│
        │ Control   │ 前压     │
        │ 信息控制  │ 抓开局   │
        └──────────┴──────────┘
```

### 5.2 CT 方核心战术详述

#### 战术 A：Conservative Retake（保守防守 / 回防架枪）

- **描述**: 开局分散防守两个包点，避免早期交火和减员；即使丢了一个包点也不慌，靠后期回防（retake）赢
- **典型执行**: 2 人守 A，2 人守 B，1 人中路自由人/游击；丢点后不急着反清，等队友集结后一起打回去
- **优势**: 不容易开局崩盘；回防时人数占优（集结后打对方残血）；稳健
- **劣势**: 容易被 T 方慢控磨掉空间；如果两边都被迫丢点就顾此失彼
- **适合场景**: 对方喜欢快攻（rush 完了没人了）、己方枪法/协同不如对方、想打后期
- **克制手段**: T 方默认慢控逐步蚕食、T 方同时打两点（爆弹）
- **所需属性**: discipline > clutch > tactics
- **失败后果**: 两边都失守，被分割围歼

#### 战术 B：Stack Site（堆人重防 / 赌点防守）

- **描述**: 开局将 3-4 人集中在某一个包点，赌 T 方正好打这个点
- **典型执行**: 3 人 A 点重防 + 1 人 B 点拖时间 + 1 人中路；如果 T 打 A 就正面接住
- **优势**: 赌对了直接团灭对方拿分；局部绝对人数优势
- **劣势**: **赌错了** = 另一个包点几乎没人，T 方轻松下包；高风险高回报
- **适合场景**: 侦察显示对方偏好打某个特定点；需要搏一把的时刻（赛点/关键局）
- **克制手段**: T 方假打转点（你堆 A 我打 B）、T 方侦察到你堆人了
- **所需属性**: firepower > tactics（赌对了就是拼枪）
- **失败后果**: 空点被下包，匆忙回防大概率输

#### 战术 C：Early Aggression（前压 / 抓开局）

- **描述**: 不在包点里等着，而是开局主动出击去中路或对方可能经过的路上拦截
- **典型执行**: 1-2 人开局就往中路/路口冲，试图在 T 方控图之前拿到首杀（opening kill）
- **优势**: 拿到首杀后立刻变成 5v4，T 方整个进攻计划被打乱；掌握主动权
- **劣势**: 压出去的人如果死了 = 包点少人；被 T 方反抓前压的人 = 直接少人防守
- **适合场景**: 己方枪法好、想打乱对方节奏、对方控图依赖某个关键选手
- **克制手段**: T 方谨慎开局先搜前压区域、T 方专门等前压的人
- **所需属性**: firepower > discipline
- **失败后果**: 开局就被打少人，整局被动

#### 战术 D：Info Control（信息控制 / 游击防守）

- **描述**: 不在任何一点死守，而是通过移动和道具来持续获取 T 方的位置信息，延迟 T 方的进攻决策
- **典型执行**: 自由人频繁换位；使用燃烧弹/烟雾弹封锁通路；不轻易交战但始终知道 T 在哪
- **优势**: 让 T 方始终不确定 CT 的部署；消耗 T 方的时间（时间对 T 是敌人）
- **劣势**: 需要极高的个人能力和地图理解；容易被 T 方抓住换位间隙
- **适合场景**: 己方有优秀的自由人选手（如 device 型角色）；对方喜欢慢控
- **所需属性**: tactics > clutch > cohesion
- **失败后果**: 被摸透了反而处处受制

### 5.3 CT 方独有情境：回防与拆弹（Retake & Defuse）

当 T 方已经成功下包之后，CT 方的任务完全改变：

#### 回防（Retake）

- **目标**: 重新夺回有炸弹的包点区域
- **关键决策**:
  - 从哪条路回？（近但危险 vs 远但安全）
  - 是否需要先清除外围的 T 方架枪人员？
  - 是否有时间一颗颗清角落？（时间够 vs 必须强排）
  - 谁去拆弹？（血量最少的人？离包最近的人？）
- **回防方式**:
  - **集体回防**: 等所有人到位后一起进（安全但可能时间不够）
  - **梯次回防**: 先进 1-2 人拖住/换人，后续跟进（冒险但争取时间）
  - **静默回防**: 尽量不被发现地靠近包点（依赖消音器/脚步控制）

#### 拆弹（Defuse）

- **基础规则**: 大拇指拆弹需 10 秒；有拆弹套件（kit）需 5 秒
- **拆弹时机**:
  - 确认周围没有敌人（或队友正在架枪掩护）
  - 注意 C4 剩余时间（还剩 7 秒才开始拆 = 来不及，除非有 kit）
  - 可以"假拆"（蹲下去假装拆引诱 T 方露头）
- **拆弹时的脆弱状态**: 拆弹时不能移动、不能射击——是完全暴露的

## 六、残局（Clutch）机制

### 6.1 什么是残局

残局指回合后期双方存活人数都较少（通常 ≤3v3，最经典的是 1v1 / 1v2 / 1v3 / 1v4）的局面。CS 的残局是最具戏剧性的时刻之一。

### 6.2 残局类型与决策框架

#### T 方残局（T 方人数 ≤ CT 方）

| 局面 | T 方目标 | T 方决策树 |
|---|---|---|
| 1v1 | 下包然后 1v1 架枪，或者直接找人对枪 | 时间够→找人对枪；时间紧→强行下包赌拆弹时间差 |
| 1v2 | 下包后利用"知道包在哪里"的信息优势 1v2 | 必须下包；下好后架枪让 2 个 CT 进来一个个处理 |
| 1v3/1v4 | 基本只能赌下包后对方犯错 | 下包→藏好→祈祷对方排错/给你机会 |

#### CT 方残局（CT 方人数 ≤ T 方）

| 局面 | CT 方目标 | CT 方决策树 |
|---|---|---|
| 1v1（未下包） | 找人杀掉 | 听声辨位→预瞄→ peek 对枪 |
| 1v1（已下包） | 安全拆弹 | 清除威胁→拆弹；注意时间 |
| 1v2（未下包） | 拖延时间 / 偷一个人 | 找机会偷一个；拖到时间结束 |
| 1v2（已下包） | 拆弹 | 需要 T 方犯错或者你打出双杀 |
| 1v3+ | 基本是绝望局面 | 赌对方集体失误 |

### 6.3 经理游戏中的残局表现

在文字驱动的经理游戏中，残局不需要模拟每一发子弹。建议用**"残局卡片"**呈现：

```text

║  🎯 残局：donk  vs  s1mple + b1t          ║
║                                          ║
║  场景：donk 已经在 A 点下包               ║
║  C4 倒计时：23 秒                         ║
║  s1mple 正从楼梯上来，b1t 在跳台架着       ║
║                                          ║
║  donk 选择：                              ║
║  ┌────────────────────────────────┐      ║
║  │ A. 架死包点（赌他们强排）       │      ║
║  │ B. 离开包点找人对枪             │      ║
║  │ C. 假装不在包点 引诱拆弹        │      ║
║  └────────────────────────────────┘      ║
╚══════════════════════════════════════════╝
```

## 七、策略碰撞矩阵（完整版）

### 7.1 T 攻 vs CT 防 — 基础碰撞表

这是 MECHANICS.md 中已有表格的**扩展和完善版本**，加入了阵营语义和更多细节。

| T 战术 \ CT 战术 | Stack 重防 | Early Agg 前压 | Conservative Retake 保守回防 | Info Control 信息控制 |
|---|:---:|:---:|:---:|:---:|
| **Rush 快攻** | 🔴 大劣（撞上重防=团灭） | 🟡 波动（看谁先杀谁） | 🟢 小优（CT 分散难挡） | 🟡 平（信息不够不影响快攻） |
| **Default 默认控图** | 🟢 优（控图后发现空点） | 🟡 波动（可能被前压打乱） | ⚪ 中性（标准的攻防博弈） | 🔴 劣（被拖太久时间不够） |
| **Slow Lurk 单摸** | 🔴 大劣（堆人的点摸不进去） | 🟢 优（前压的人身后空了） | 🟡 平（有机会但风险高） | 🟢 优（CT 到处跑容易偷到） |
| **Fake Rotate 假打** | 🟢🟢 **大优**（堆错点了=空点） | 🟡 小劣（前压的人不一定被骗） | ⚪ 中性（CT 不急换防就没用） | 🔴 大劣（信息控制=不容易被骗） |
| **Late Execute 拖后执行** | 🟢 优（最后时刻 CT 没时间换防） | 🟡 平（前压早结束了） | 🟢 优（CT 稳守但你信息充分） | 🔴 劣（被拖到自己也没时间了） |

**图例**:
- 🟢🟢 大优 (+6~8 修正): 该组合显著有利于攻击方
- 🟢 优 (+3~5 修正): 有利于攻击方
- ⚪ 中性 (±0~2): 标准博弈，看执行
- 🟡 波动 (-3~+3): 结果高度依赖于具体执行和运气
- 🔴 劣 (-3~-5 修正): 有利于防守方
- 🔴🔴 大劣 (-6~-8 修正): 显著有利于防守方

### 7.2 碰撞结果的修正因素链

```
最终结果 = 基础碰撞值
         + T 方对应属性的加权值
         - CT 方对应属性的加权值
         + 装备质量修正（经济等级差）
         + 化学反应修正
         + 特质触发修正
         + 侦察/记忆修正
         + 教练哲学修正
         ± 随机因子（缩小范围后）
```

## 八、阵营转换：下半场交换

### 8.1 交换 sides

一场完整的 CS 比赛（在现实中）是 MR12 或 MR15 格式，即一方先打 12/15 局 T 方，然后**双方交换阵营**，再打 12/15 局。

### 8.2 在经理游戏中的简化

CS Cup Manager 使用的是 BO5 压缩回合制（5 局 3 胜）。建议的处理方案：

**方案 A — 固定阵营（推荐用于 v0.1）**:
每场比赛随机/交替决定玩家队伍是 T 还是 CT，整场比赛保持不变。
- 优点：简单，UI 和逻辑清晰
- 缺点：缺少交换阵营的策略深度

**方案 B — 中途交换（推荐用于 v0.2）**:
BO5 中第 3 局结束后交换阵营（前 2 局 T，后 2 局 CT；或反之）。
- 优点：体现 CS 的完整体验，增加策略变化
- 缺点：实现稍复杂

**方案 C — 每局随机（不推荐）**:
每局随机 T/CT。
- 缺点：不符合 CS 比赛惯例，玩家会觉得怪异

### 8.3 交换阵营的战略意义

在真实比赛中，交换阵营意味着：
- 地图通常对某一方略有利（比如 Nuke 的 T 方优势、Train 的 CT 方优势）
- 前半场的情报（对方的习惯、热区、弱点）在后半场仍然有价值
- 有些选手/队伍明显更擅长某一方的打法

经理游戏中可以通过**"地图倾向"**参数来体现这一点（见第十章）。

## 九、地图与位置概念

### 10.1 通用地图术语

| 术语 | 英文 | 含义 |
|---|---|---|
| 包点 | Bombsite (A/B) | 安放/拆除 C4 的目标区域 |
| 中路 | Mid | 连接两个包点和双方出生点的中央通道 |
| 连接/通道 | Connector / Link | 包点之间的通路 |
| 斜坡/高台 | Ramp / Upper | 高低差位置 |
| 狙位 | AWP Angle / Nest | 适合 AWP 架枪的位置 |
| 预瞄位 | Pre-fire / Common Angle | 最可能遇到敌人的位置 |
| 死角 | Dead Angle | 视线无法覆盖的区域（常被 lurk 利用） |
| 回防路线 | Rotation Route | CT 从一个包点到另一个包点的内部路径 |
| 外场 | Outside / Heaven | 某些地图的外部区域 |

### 10.2 位置对战术的影响

虽然经理游戏不需要渲染 3D 地图，但**位置概念直接影响战术选项的有效性**：

- **狭窄通道**（如 Mirage 的 B 点二楼窗口）：Rush 有效（人多硬冲），Lurk 很难（没地方躲）
- **开阔区域**（如 Dust2 的 A 大道）：Info Control 有效（视野好），Rush 危险（容易被狙）
- **多通路包点**（如 Inferno 的 A 点）：难以堆防，Default 控图效果更好
- **单一入口包点**（如 Train 的 A 点）：堆防效果好，Rush 很难打进去

**建议**: 为每张地图添加 `mapProfile` 参数，微调各战术的基础修正值：

```typescript
// 示例数据结构（供实现参考）
interface MapProfile {
  name: string;              // 如 "Mirage", "Dust2", "Inferno"
  tSideBias: number;         // 正值=T方优势，负值=CT方优势 (-3 ~ +3)
  tacticalModifiers: {
    rush: number;            // 本图 rush 加成/减成
    default: number;
    lurk: number;
    fake: number;
    lateExecute: number;
  };
}
```

## 十、教练视角的决策分层

### 10.1 经理游戏中的决策不等同于选手操作

**关键区分**：玩家扮演的是教练/经理人，不是扣动扳机的选手。因此决策应该是**战术层面**而非**操作层面**。

| ❌ 操作层决策 | ✅ 战术层决策 |
|---|---|
| "向左预瞄" | "进攻 A 点" |
| "扔闪光后冲出去" | "假打转 B 点" |
| "这个夹角架枪" | "保守回防拿后期" |
| "切刀跑快一点" | "ECO 保枪打下一局" |

### 10.2 完整回合决策流（按阵营，含经济约束）

│          [甲+拆弹套件(CT)] [纯省留钱]                     │
│    影响：防具水平、道具数量、拆弹能力                     │
│                                                          │
│  Step 2: 战术决策                                        │

### 10.3 玩家可做的决策总览表

按照真实 CS 回合分层，玩家在所有可能的时刻会面对以下决策：

#### A. 开局决策（每局都有，第 1 局特殊）

| 回合 | 经济决策 | 战术决策（T） | 战术决策（CT） |

| **第 1 局** | 手枪局购买（强制） | Rush / Default / Lurk / Fake / Late Execute | Stack / Aggro / Retake / Info Ctrl |

| **Bonus Round** | 是否省钱留装备 | 同上 | 同上 |

| 触发条件 | 决策 |

| 暂停窗口触发 | 战术重置 / 情绪重置 / 纪律重置 |

## 十一、常见设计陷阱与反模式

### 11.1 已知问题（当前代码中存在的）

| # | 问题 | 为什么是错的 | 正确做法 |
|---|---|---|---|
| 1 | 第 1 局不是手枪局 | 真实 CS 第 1 局双方 $800 只能手枪 | 第 1 局强制手枪局，第 2+ 局才由经济决定装备 |
| 2 | T/CT 用同一套选项 | 违背 CS 核心不对称性 | 根据阵营动态切换选项集 |
| 3 | 经济自由选择 | CS 的经济是累积的，不能凭空选全买 | 金钱追踪 + 连败奖励 + 击杀奖励决定可用等级 |
| 4 | 每场只有一个决策点 | 经理参与感极低 | 每回合至少 1 次经济+1 次战术决策 |
| 5 | 随机范围 ±9 淹没策略差异 | 决策无意义 | 缩减到 ±3~4，让策略碰撞主导结果 |
| 6 | 不追踪回合间经济 | 无法体验连败恢复/Bonus Round/强起赌博 | 实现完整经济状态机（见第三章） |

### 11.2 设计时应避免的反模式

| 反模式 | 说明 |
|---|---|
| **对称复制** | 把 T 方的选项换个名字给 CT 方用——两者的决策逻辑完全不同 |
| **操作粒度过细** | 让玩家选择"谁去哪""扔什么道具"——这是选手的事，不是教练的事 |
| **忽略时间维度** | 所有战术看起来一样紧迫——CS 的精髓在于"快vs慢"的时间博弈 |
| **经济无感** | 每一局都能全买——失去了 CS 经济循环的核心乐趣 |
| **无阵营叙事** | 无论 T 还是 CT，事件卡读起来一模一样——浪费了不对称性带来的叙事潜力 |
| **跳过手枪局** | 直接从全买开始——第 1 局的 $800 手枪博弈是整个比赛节奏的基石 |
