# 编年史名场面文本库 — CS Cup Manager

## 概述

编年史 (Chronicle) 是三年赛季结束后的总结界面，包含两个核心叙事组件：

1. **名场面 (starHighlight)**: "被记住的一回合"——比赛中最精彩的操作瞬间
2. **关键一战 (definingCup)**: "定义性杯赛"——三年中最重要的杯赛及其意义

---

## 名场面池 (starHighlight)

### 从 8 → 22 模板
从 8 条扩展至 22 条，覆盖更多 CS 电竞经典场景：

| 场景类型 | 数量 | 典型描述 |
|----------|------|---------|
| 扫射转移/连杀 | 2 | 一个弹匣完成三杀，解说声音到第三个才来得及喊出来 |

| 极限操作 | 2 | 最后一匣点射三杀、经济局捡枪翻盘 |

### 设计原则
- 每个模板都有具体的 CS 操作细节（预瞄、弹道、假拆、混烟…）
- 不只是描述"他杀了三个人"，而是描述**为什么能杀三个**
- 引入观众/解说反应，增强名场面的仪式感
- 保持变量占位符（{}），由系统填入真实选手名和位置

### 示例
```
"{star_player_name} 在混烟里听到了拆包声——就那一丝。他没有扫满弹匣，
而是先压三发逼停拆包，等对手重新碰包时穿点穿中。观众席上有人以为要输了，
然后听到击杀提示才反应过来赢了。"
```

## 关键一战池 (definingCupWhyPool)

### 6 种叙事类型

| ID | 触发条件 | 叙事主题 | 文本数 |
|----|---------|---------|--------|
| defining-comeback | 亚军 | 绝地翻盘——从落后到夺冠 | 3 |
| defining-dynasty | 冠军 + 2次以上夺冠 | 王朝落成——统治力的确立 | 3 |
| defining-breakthrough | 冠/亚 + 1次以下夺冠 | 首次突破——从黑马到竞争者 | 3 |
| defining-rivalry | 冠/亚/四强 | 宿敌对决——打败最强对手 | 2 |
| defining-grind | 四强或更低 | 磨砺之路——留下训练笔记 | 2 |
| defining-first-cup | 冠军 | 第一座奖杯——起点记忆 | 1 |

### combined 总变体: 14 条

### 选择逻辑
- 根据 `definingCup.placement`（名次）和 `cupWins`（总冠军数）匹配
- 同一条件下随机选一条 `texts[]` 变体
- 占位符 `{cupName}`, `{placement_name}`, `{star_name}` 由系统填入

### 示例

**绝地翻盘 (defining-comeback)**:
```
"{cupName} 的决赛打到第三张图时，Team gun 还落后 4 分。没有人觉得能翻
——除了坐在服务器里的五个人。一回合、再一回合，他们把比分追平，然后反超。
夺冠的那一刻，场馆安静了一秒才炸开：不是因为意外，是因为没人相信真的发生了。"
```

**王朝开始 (defining-dynasty)**:
```
"{cupName} 是 Team gun 王朝落成的一站。在这之前，人们说他们是强队；
在这之后，人们开始讨论他们的统治力。不是因为赢了一站杯赛，而是因为赢的方式
——从小组赛到决赛，没有丢掉过一张关键图。"
```

## MVP 评语池 (pickDefiningCupEncouragement)

根据杯赛名次和冠军数，为最具价值选手生成个性化评语：

| 条件 | 评语主题 | 变体数 |
|------|---------|--------|
| 冠军 + 王朝 | 关键残局大师，对手研究焦点 | 2 |
| 冠军 | 决赛carry，奖杯回应质疑 | 2 |
| 亚军 | 差一步的底色，集锦反复出现 | 2 |
| 其他 | 值得被记住的表现 | 2 |
