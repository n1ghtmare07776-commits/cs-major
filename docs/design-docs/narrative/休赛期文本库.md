# 休赛期文本库 — CS Cup Manager

## 概述

休赛期事件池 (`offseasonEventPool`) 用于赛季之间的过渡阶段。事件在赛季总结界面以回顾卡片形式展示，记录休赛期发生的关键事件和处理结果。

### 事件统计
- **总计**: 46 个事件
- **Tone 分布**: positive 19 / neutral 14 / negative 13
- **Severity 分布**: mild 37 / severe 9
- **事件选择逻辑**: `pickFreshEvent()` 基于 seed + 历史记录避免重复

## 叙事设计原则 (同 v1.0)

### 1. 真实感优先
- 所有叙事必须来自真实的 CS 电竞生态：选手日常、教练决策、俱乐部运营
- 避免"戏剧化"编造——休赛期的戏剧性来自日常中的微小张力
- 对话和细节要像记录片而非剧本

### 2. 选择逻辑性
- 每个事件的 `choice` 是教练在真实情况下会做出的决策
- 不追求"英雄式"选择，而是"合理且专业"的选择
- 牺牲清晰标注在 delta 中，让玩家理解每个选择的代价

### 3. 结果因果链
- `result` 必须准确反映 `choice` 的后果
- 正面选择有代价（如海外集训 → 耗预算 / 牺牲火力）
- 负面选择有修复（如版本更新 → 短期阵痛 / 长期战术提升）

### 4. 语气分层
- **Positive**: 建设性事件，不意味着无代价
- **Neutral**: 中性事件，好坏取决于选择
- **Negative**: 挑战性事件，但总有缓解方案

## 属性影响规范

| 属性 | 含义 | 典型增幅 | 典型降幅 |
|------|------|---------|---------|
| firepower | 个人枪法/状态 | +1~3 | -1~2 |
| tacticalExecution | 战术执行/准备度 | +2~5 | -1~3 |
| cohesion | 团队默契/配合 | +1~6 | -1~3 |
| discipline | 纪律/专注力 | +1~4 | -1~2 |

### 影响分配原则
- Mild 事件总影响幅度：3~7 点
- Severe 事件总影响幅度：5~10 点
- 单一属性最大增幅不超过 +6，最大降幅不超过 -3
- 每赛季约触发 4-7 个休赛期事件，累计影响在可控范围内

## 事件选择与防重复

```javascript
function pickFreshEvent(pool, seed, history, lane) {
  // 1. 优先选择从未出现过的
  // 2. 其次选择最近6次未出现的
  // 3. 再次选择近12次出现少于3次的
  // fallback: 随机选择
}

// 赛季初过滤：第一个事件不允许太正面的 mild 事件
// 保证休赛期叙事以"面对现实"开场
```
