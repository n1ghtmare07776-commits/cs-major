# 比赛决策选项完整规范

> **目标读者**: UI 实现 agent / 游戏逻辑 agent
> **用途**: 比赛进行时，每个回合根据阵营、经济、回合类型动态生成决策选项。
> **原则**: 不同阵营看到完全不同的选项集，没有混用。不同经济等级锁住不可用的战术。
> **版本**: v1.0 | 2026-06-26

---

## 零、当前问题诊断

```
当前 UI（截图所示）:

  标题: "默认架枪还是爆弹抢点"  ← 混用了CT("架枪")和T("爆弹")的术语
  选项A: 快攻包点              ← T 方战术
  选项B: 默认架枪              ← CT 方战术（"架枪"是防守动作）
  选项C: 假打转点              ← T 方战术

  问题:
  1. 玩家是 T 方 → "默认架枪" 不该出现
  2. 玩家是 CT 方 → "快攻包点""假打转点" 不该出现
  3. 只有 3 个选项太单薄，没有体现经济约束和情境变化
  4. 手枪局、ECO 局、残局、下包后——全部一样的选项
```

**正确做法**：5 大类 × 阵营 = 每回合玩家看到 3-5 个**高度贴合的选项**。

## 二、手枪局独有选项（第 1 局 / 下半场第 1 局）

**手枪局不按常规战术分类**——双方 $800，买不起步枪。核心决策是**钱怎么花**。

### 2.1 手枪局通用选项

| # | 选项 ID | UI 主文案 | UI 副标题 | 花费 | 效果 |
|---|---------|----------|----------|------|------|
| P1 | `pistol_full_armor` | 🛡️ 全甲手枪 | $650 买甲，剩下的买 USP/Glock | $650 | 防具满，火力基础 |
| P2 | `pistol_half_armor_nade` | 💣 半甲+道具 | $300 甲 + $200 闪光 + 留钱 | $500 | 有道具配合，甲稍薄 |
| P3 | `pistol_deagle_gamble` | 🎲 沙鹰豪赌 | $700 买 Deagle，不买甲 | $700 | 一枪爆头秒杀，但没甲=脆 |
| P4 | `pistol_save_money` | 💰 省钱等下局 | 什么都不买，初始手枪出门 | $0 | 赢了经济极好，输了正常 |

### 2.2 CT 方手枪局独有

| # | 选项 ID | UI 主文案 | UI 副标题 | 花费 |
|---|---------|----------|----------|------|
| PC1 | `pistol_kit` | 🔧 甲+拆弹套件 | $650 甲 + $400 拆弹套件 → 5 秒拆弹 | $1050 |

> **注意**: $800 不够全甲($650) + 拆弹套件($400) = $1050，实际可用组合：

> - 半甲($300) + 拆弹套件($400) → 剩 $100

**手枪局后无缝衔接**到常规回合经济系统（见第三章）。

## 六、暂停（Timeout）选项

全场仅一次。触发条件：对手连赢 ≥ 2 回合，或己方选手心态/纪律异常。

| TO2 | `timeout_emotional` | 🫂 情绪重置 | 远离屏幕深呼吸，冷静下来 | morale +5, condition +4 |
| TO3 | `timeout_discipline` | ⚔️ 纪律重置 | 停止浪和送，回到默认 | discipline +6, tactics +2 |

## 七、完整回合决策流（按引擎状态机）

### 7.1 每个回合的决策生成算法

```
function getRoundOptions(state):
  1. 判断阵营: T / CT
  2. 判断回合类型:
     - 手枪局? → 返回 §2 选项集
     - 正常局? → 继续
     - 下包后? → 返回 §4 选项集
     - 残局?   → 返回 §5 选项集
  3. 计算当前经济等级: Lv1 ~ Lv5
  4. 查 §3 选项锁 → 过滤出本阵营可用选项
  5. 检查是否触发暂停条件 → 是则先弹出 §6
  6. 返回最终选项列表（3-5 个）

  输出: {
    options: DecisionOption[]  // 3-5 个
    side: "T" | "CT"
    economy: "pistol" | "eco" | "weakbuy" | "forcebuy" | "fullbuy"
    isPostPlant: boolean
    isClutch: boolean
  }
```

## 八、UI 实现规范

### 8.1 选项卡片结构

```typescript
interface DecisionOption {
  id: string;            // 如 "rush", "default", "lurk"
  label: string;         // 主文案，如 "爆弹抢点"
  sublabel: string;      // 副标题，如 "五人同时冲一个包点"
  icon: string;          // emoji 图标
  tone: "aggressive" | "balanced" | "cautious" | "gamble";
  difficulty: "easy" | "normal" | "hard";
  requires: {            // 关键属性提示
    stat: string;        // 属性名
    min: number;         // 建议最低值
  }[];
  economyLabel?: string; // 经济提示，如 "需要 Lv3+"
  locked: boolean;       // 是否被经济锁住
  lockReason?: string;   // 锁住原因，如 "经济不足"
}
```

### 8.3 选项之间的差异化

| 节奏 × 风险 | 激进 | 均衡 | 保守 | 赌命 |
|------------|------|------|------|------|
| **快** | 🔥 爆弹抢点 | ⚡ 前压抓人 | — | 🎲 赌命快攻 |
| **慢→快** | ⏳ 拖后执行 | — | — | — |
| **慢** | 🎯 赌点重防 | 🧊 默认控图 | 🛡️ 保守回防 | 🎲 赌命堆点 |
| **极慢** | 👻 单摸夹击 | 🕵️ 信息控制 | 💰 保枪省钱 | — |

- **8 个正常选项覆盖了博弈空间的所有维度**，不会出现"三个选项实际上是一回事"
- ECO 局 2-3 个选项（保枪 vs 赌命 + 可能的慢控）
- 满买局 4-5 个选项

## 十、与现有事件系统的桥接

### 10.1 赛中决策卡 vs 赛中事件卡

| 概念 | 决策卡 | 事件卡 |
|------|--------|--------|
| **用途** | 玩家**选择战术/经济** | 玩家**应对突发状况** |
| **时机** | 每回合必定出现 | 概率触发（条件满足） |
| **选项数** | 3-5 个 | 2-4 个 |
| **来源** | 本文件定义的选项集 | `mechanic-events.ts` / `events.ts` |
| **玩家感受** | "我选择怎么打" | "事情发生了，我怎么应对" |

### 10.2 回合时序中的位置

```
 第 N 局开始
    │
    ├── 1. 经济累积（引擎自动）
    │      $800 手枪局 / 按上局结果累加
    │
    ├── 2. 决策卡弹出 ★（本文件的选项）
    │      玩家选择战术
    │
    ├── 3. 事件卡可能触发（概率）
    │      选手心态崩了 / 化学反应触发 / 高风险戏剧
    │
    ├── 4. 碰撞计算（引擎）
    │      T 战术 vs CT 战术 → 查碰撞表 → 结算
    │
    ├── 5. 结果叙事
    │      "donk 在 A 点拿下首杀，队伍 5-0 团灭对手"
    │
    └── 6. 如果触发残局 → 残局决策卡弹出
           玩家做最后决策 → 最终结算
```

## 十一、反模式警告（实现时避坑清单）

| # | 错误做法 | 为什么错 | 正确做法 |
|---|---------|---------|---------|
| 1 | T/CT 混合显示选项 | 违背 CS 核心不对称性 | 按阵营动态过滤 |
