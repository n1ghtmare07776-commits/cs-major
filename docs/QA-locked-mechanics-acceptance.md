日期：2026-06-26  
角色：QA / 验收子代理  
范围：基于当前仓库状态，更新“锁定机制”验收口径，重点覆盖：

1. 三年全流程是否真的能跑完
2. 单场比赛是否稳定落在 `10-15` 张事件卡、玩家决策 `<= 5`
3. 经济 / 战术记忆 / 暂停 / 长期默契 / 交易限制是否真的“可验”
4. UI / 文案常见错误项

本清单不是未来设计愿景，也不是只看文档口号。通过标准只有三个：

- 状态或规则真实存在于实现中
- 玩家主路径里真的会用到
- 自动化或手工游玩能证明它有效

## 三、按最终目标重排后的验收主清单

## 1. 三年全流程验收

### 目标口径

必须能从征召室一路跑到第三年编年史，完整覆盖：

- `3` 个赛季
- 每赛季 `3` 杯
- 共 `9` 杯
- 杯间 hub
- 年度奖项
- 最终编年史

### 自动化通过条件

自动化至少要证明：

- 确认阵容后进入 `seasonIndex = 1`
- 最终能抵达 `screen === "chronicle"`
- `cupRecords.length === 9`
- `seasonRecords.length === 3`
- 杯赛顺序始终是 `Katowice -> Cologne -> Major`
- `awards -> between-cups -> bracket`
- `awards -> annual-awards -> bracket/chronicle`
  这些关键切屏没有死路

### 当前仓库证据

`tests/app/browser-flow.test.mjs` 已经写了全流程黄金路径断言，目标状态包括：

- `screen === "chronicle"`
- `campaign.seasonIndex === 3`
- `campaign.cupRecords.length === 9`
- `campaign.seasonRecords.length === 3`

### 当前判定

- **口径上：已锁住**
- **仓库现状上：被 app 测试语法错误暂时阻断**

### 手工验收要点

至少完整打穿 1 个存档，并逐段确认：

- 第 1 年第 3 杯结束后进入年度奖项，而不是直接跳下一杯
- 第 3 年结束后进入编年史，而不是重新开档
- 任意一杯八强出局后，后续杯赛和年份仍可继续
- `重新开始` 不会误替代 `进入下一阶段`

## 2. 单场比赛节奏验收

### 目标口径

单场玩家比赛应保持：

- 总事件卡 `10-15`
- 玩家决策卡 `<= 5`
- 默认体验目标约 `3` 张决策卡

### 自动化通过条件

自动化至少要断言：

- `cards.length >= 10`
- `cards.length <= 15`
- `decisionCards.length <= 5`
- 存在足量非决策 narrative 卡
- 开局经济选择与开局战术选择是两张不同的决策卡

### 当前仓库证据

`tests/app/browser-flow.test.mjs` 已明确断言：

- 事件卡数量在 `10-15`
- 决策卡数量在 `3-5`
- 非决策卡数量至少 `5`

### 当前判定

- **已锁住**

### 手工验收要点

不要只抽查第一场。至少检查：

- 第一赛季第一杯
- 半决赛或决赛
- 第二或第三赛季任一杯

要确认没有发生以下漂移：

- 后续加戏把事件卡堆到 `16+`
- 每张卡都要点按钮，决策密度失控
- 文案说“约 10 张”，结果真实体验只剩 7-8 张

## 3. 锁定机制“是否真的可验”

这一部分是本清单的核心。重点不是“有没有 helper”，而是“有没有接进主路径”。

### 3.1 半数值经济

#### 最终目标

必须同时满足：

- 后台有真实 economy 状态
- 前台主要显示 `全起 / 半起 / 强起 / ECO / 奖励局`
- 输赢会改变下一回合买枪层级
- 玩家能感受到“这局穷”“这局能强起”“这局保枪赚到了”

#### 当前仓库证据

已存在的内容：

- `describeEconomyPath()`
- `createEconomyState()`
- `settleEconomyRound()`
- `BUY_LABEL`
- `tests/app/mechanics-locks.test.mjs` 已锁住经济 helper 的规则期望

但当前主路径缺口很明确：

- 真实比赛状态没有把经济流转接入事件推进
- 开局 `全起 / 半甲控图 / ECO` 的选择主要只改变当前卡的 `score` 和 `delta`
- active match 没有把后续每张卡按经济层级持续演进
- 玩家 UI 也没有展示每个阶段的买枪层级变化

#### 当前判定

- **部分锁住，仍不可通过最终验收**

更准确地说：

- **helper 已锁住**
- **玩家主路径未锁住**

#### 必须补的自动化

至少新增一条集成级测试，证明：

- 开局选 `ECO` 与 `全起` 会进入不同后续经济路径
- `输第一局 -> 第二局通常更穷`
- `连输两局 -> 第三局恢复买枪能力`
- `赢两局后输一局 -> 第四局仍保有可买枪层级`
- 玩家比赛中真实可见这些变化

---

### 3.2 中等战术反制 / 战术记忆

#### 最终目标

必须同时满足：

- 对手会读玩家最近 `2-3` 次重复开局
- 这种反制强度是中等，不是秒杀
- 玩家能靠变招、假打、侦察、暂停缓解
- 文本会解释“为什么被读到了”

#### 当前仓库证据

已存在的内容：

- `summarizeReadPressure()`
- helper 测试已断言：
  - 第 1 次是 `fresh`
  - 连续第 2 次是 `warming`
  - 连续第 3 次是 `read`
- `tacticEdgeForChoice()`、`chooseOpponentPlan()` 已体现设计方向

但当前主路径缺口同样明显：

- `strategyMemory.recentOpenings` 没有在真实比赛后被更新
- active match 没有把最近开局历史灌进真实 `readPressure`
- `resolveFeaturedMatch()` 的最终胜负边只看：
  - 队伍强度差
  - 当前比分势能
  - 累积 impact
  - 是否用过暂停
- 当前“被读到”更多还是文案意图，不是比赛中持续生效的状态

#### 当前判定

- **部分锁住，仍不可通过最终验收**

同样要拆开看：

- **helper 已锁住**
- **玩家主路径未锁住**

#### 必须补的自动化

至少新增一条集成级测试，证明：

- 连续 2-3 次同类开局后，对手反制确实升高
- 切到假打或变招后，惩罚会回落
- 这种回落会影响结果分布，不只是文本提示

---

### 3.3 中暂停

#### 最终目标

中暂停必须满足：

- 先解释为什么要停
- 再提供 `战术 / 情绪 / 纪律` 三选一
- 玩家一场只能用一次
- 暂停后应真实影响后续局段，而不只是本卡加分

#### 当前仓库证据

当前已经具备：

- 独立“暂停窗口”卡
- 三选一结构
- `timeoutUsed` 状态
- UI 会显示“暂停仍可使用 / 暂停已使用”
- 对应测试与页面路径能稳定触发

当前不足：

- 暂停效果主要写在当前卡的 `score / delta / playerDelta`
- 最终结算只给 `timeoutUsed` 一个很轻的结果修正
- 没有明确的“暂停后持续改变后续战术读法 / 情绪状态 / 纪律曲线”

#### 当前判定

- **部分锁住**

这是四类里最接近“已接线”的一项，但还没到最终可验标准。

#### 必须补的自动化

至少新增一条测试，证明：

- 三种暂停不是同一 buff 换皮
- 选择后至少影响 1 个后续事件或后续结算维度
- 使用后不会再次出现有效暂停入口

---

### 3.4 轻度长期默契成长

#### 最终目标

必须同时满足：

- 同一首发打正式比赛会累计隐藏默契
- 增幅轻，不会滚雪球失真
- 玩家能感觉更顺，但看不到完整公式
- 换人后默契积累会中断或折损

#### 当前仓库证据

已存在的内容：

- `continuityBonusForMatches()`
- helper 测试已锁住轻度成长与上限封顶
- campaign state 里已经有 `lineupMatches`

但当前主路径存在关键缺口：

- 没有看到正式比赛后递增 `lineupMatches`
- 当前实时 stats 计算并没有把这条成长稳定灌进玩家比赛
- 也没有任何 UI 或结果文本能让玩家感知“这套首发磨出来了”

#### 当前判定

- **部分锁住，仍不可通过最终验收**

准确说：

- **helper 已锁住**
- **长期经营路径未锁住**

#### 必须补的自动化

至少新增一条测试，证明：

- 同一首发打更多正式比赛后，战术执行或稳定性有轻度提升
- 换人后提升被打断
- 上限不会失控
- UI 不会直接把完整公式暴露给玩家

---

### 3.5 每杯间最多一次交易

#### 最终目标

必须满足：

- 每个杯间最多 1 次主动交易尝试
- 交易受资金、报价、队伍处境、球员意愿共同影响
- 失败只反馈 1-2 条主因

#### 当前仓库证据

当前浏览器主路径里：

- 有杯间 hub
- 没有真实交易入口
- 没有交易次数状态
- 没有谈判结果状态

#### 当前判定

- **未锁住**

#### 必须补的自动化

至少新增：

- 一条“杯间只能尝试一次交易”的路径测试
- 一条“失败给出 1-2 条主因”的反馈测试

## 4. UI / 文案常见错误验收

这一类问题不会直接改变模拟层，但会明显破坏“像最终版”的感觉，必须单独验。

### 4.1 必查拼写与术语

必须持续防回归：

- `默认架枪`，不能写成 `默认夹枪`
- `四分之一决赛 / 半决赛 / 决赛 / 冠军` 标签正确
- `Katowice / Cologne / Major` 顺序正确
- `5 首发 + 1 替补` 口径前后一致

当前自动化已覆盖：

- `架枪` typo
- bracket tree 关键标签

### 4.2 必查状态文案和真实状态一致

常见错误项：

- UI 说“暂停已使用”，但逻辑上还能继续用
- UI 说“三年征途”，实际流程只跑到一杯
- UI 说“约 10 张事件框”，实际卡数漂移明显
- KDA 面板“阵亡 / 存活”与最新事件不同步

### 4.3 必查占位和假完成文本

必须防止：

- 英文占位缩写
- `待定`
- 缺失选手名
- 颁奖页或编年史出现空字段
- 杯间页只剩设计说明式文案，像 debug 页面

允许 `待定` 的场景只有一种：

- bracket 中未来轮次尚未决出时的占位

一旦轮次已结算，相关位置就不能继续保留占位词。

### 4.4 必查“说明型文案替代结果文案”

要避免这些最终体验错误：

- 用“这部分保持轻量”之类开发说明文本代替真正赛况
- 用设计口径介绍替代实际结果反馈
- 让奖励 / 年度总结 / 编年史看起来像文档节选，而不是游戏结果页

## 四、建议新增或升级的验收测试

### P0：修复 app 测试启动阻断

### P1：三年全流程补强测试

在现有 full campaign 测试基础上再补：

- 杯赛顺序断言
- 每次 `advance` 后 screen 迁移断言
- 八强出局后的后续年份仍能继续

### P1：多场比赛节奏回归测试

## 五、当前最关键的 5 个验收点

1. **三年九杯必须真的跑到编年史**
   这条现在已经有目标测试，但当前被 app 语法错误卡住，仍属 release blocker。

2. **每场比赛必须稳住在 `10-15` 张事件卡、决策 `<= 5`**
   这是当前最明确、也最容易后续回归漂移的体验护栏。

3. **经济必须从 helper 变成主路径状态**
   现在已经有规则函数，但还不足以证明玩家真的在打“半数值经济”。

4. **战术记忆 / 反制必须从文案意图变成实际结果分布**
   如果这一条不接通，侦察、变招、暂停都会显得像包装层。

5. **长期经营机制必须至少接通一条真实长期反馈链**
   当前长期默契与交易限制都还没有真正进入可验状态，三年模式会因此偏“能跑完”，但不够“像经理游戏”。

## 六、结论

基于当前仓库状态，这份更贴近最终目标的判断是：

- **已锁住**：
  - 三年全流程目标路径（但被 app 测试语法错误暂时阻断）
  - 单场 `10-15` 张事件卡
  - 单场玩家决策 `<= 5`
  - bracket / KDA / typo 的部分 UI 文案护栏

- **部分锁住**：
  - 半数值经济
  - 中等战术反制
  - 中暂停
  - 轻度长期默契成长

- **未锁住**：
  - 每杯间最多一次交易

因此，当前最正确的验收口径不是“这些机制都做完了”，而是：

- **流程和比赛节奏护栏已经开始成形**
- **长期经营与可验证机制层仍处在 helper 先行、主路径未完全接线的阶段**

后续放行应严格按“真实状态 + 主路径调用 + 自动化证明”三件套执行。
